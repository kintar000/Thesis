Adding New Assets

  1. Navigate to Asset Type
    Click on the appropriate section (Assets, Components, etc.) from the sidebar menu.
  2. Click "Add New"
    Look for the "Add Asset" or similar button, usually located at the top right of the page.
  3. Fill Required Information
    Complete all required fields marked with an asterisk (*). Common fields include:
    • Name/Model
    • Asset Tag (auto-generated or custom)
    • Serial Number
    • Category/Manufacturer
    • Purchase Date and Cost
    • Location
  4. Set Initial Status
    Choose the appropriate status: Available, Checked Out, Maintenance, etc.
  5. Save the Asset
    Click "Save" to create the asset. You'll receive a confirmation message.
üí° Asset Tag Best Practice: Use a consistent naming convention for asset tags, such as "DEPT-TYPE-NUMBER" (e.g., IT-LAP-001 for IT department laptop #001).

Asset Checkout Process

When someone needs to use equipment temporarily:

  1. Find the Asset
    Use search or browse to locate the asset you want to checkout.
  2. Click "Checkout"
    This button appears in the asset's action menu or details page.
  3. Select User
    Choose who will receive the asset. You can search by name or username.
  4. Set Checkout Details
    Enter:
    • Expected return date
    • Purpose or project notes
    • Special handling instructions
  5. Confirm Checkout
    Review details and click "Checkout Asset" to complete the process.

Asset Check-in Process

When equipment is returned:

  1. Locate Checked-Out Asset
    Find the asset in the system (it will show as "Checked Out" status).
  2. Click "Check In"
    This option appears in the asset's action menu.
  3. Inspect Condition
    Physically examine the asset for:
    • Physical damage
    • Missing accessories
    • Functionality issues
  4. Update Asset Information
    Record:
    • Current condition
    • Any issues found
    • New location if changed
  5. Complete Check-in
    Click "Check In Asset" to make it available for others.

5. üì° Monitoring & Discovery Systems

Virtual Machine Monitoring

SRPH-MIS integrates with Zabbix to provide real-time monitoring of virtual machines.

Setting Up VM Monitoring

  1. Access VM Monitoring
    Navigate to "Monitoring" ‚Üí "VM Monitoring" from the sidebar.
  2. Configure Zabbix Connection
    Click the "Settings" tab and enter:
    • Zabbix URL (e.g., http://107.105.168.201/zabbix)
    • API Key (generated from Zabbix Administration panel)
    • Sync interval (recommended: 5 minutes)
  3. Test Connection
    Click "Test Connection" to verify the API key and URL are correct.
  4. Enable Auto-sync
    Turn on automatic synchronization to keep VM data current.

Monitoring Capabilities

üñ•Ô∏è VM Status

Real-time status tracking (Running, Stopped, Error)

‚ö° Performance Metrics

CPU usage, memory consumption, disk space

Network Discovery

Automatically discover and inventory network devices on your infrastructure.

Running Network Scans

  1. Access Network Discovery
    Go to "Monitoring" ‚Üí "Network Discovery" in the sidebar.
  2. Configure Scan Parameters
    Set up your scan with:
    • Target subnets (CIDR format: 192.168.1.0/24)
    • Port ranges (default: 22,80,443,3389)
    • DNS servers (default: 107.105.134.9, 107.105.134.8)
    • Timeout and retry settings
  3. Start Discovery Scan
    Click "Start Scan" and monitor progress in real-time.
  4. Review Results
    Examine discovered devices, their services, and configurations.
  5. Generate Reports
    Export discovery results for documentation and analysis.
‚ö†Ô∏è Security Consideration: Network discovery scans should be performed during maintenance windows when possible. Ensure proper authorization before scanning production networks.

Discovery Results

The network discovery process identifies:

  • Device Information: IP addresses, hostnames, MAC addresses
  • Operating Systems: Detected OS types and versions
  • Open Services: Running services and open ports
  • Hardware Details: When available, CPU, memory, and storage info
  • Software Inventory: Installed applications and configurations

BitLocker Recovery Keys

üîí Security Critical Feature: BitLocker recovery key management is restricted to authorized security personnel only. Access requires administrator privileges.

Key Management Functions

  • Secure Storage: Centralized storage of recovery keys
  • Asset Association: Link keys to specific devices and users
  • Access Logging: Complete audit trail of all key access attempts
  • Emergency Access: Quick key retrieval for urgent situations
  • Search Capabilities: Find keys by serial number or device identifier

Recovery Process

  1. Verify user identity and authorization level
  2. Locate the device in the BitLocker Keys module
  3. Retrieve the associated recovery key
  4. Provide the key to the authorized user
  5. Log the key access event for audit purposes
  6. Monitor for successful drive recovery

8. üîß Troubleshooting & Common Issues

Login & Authentication Issues

üö´ Can't Log In

Symptoms: Login form rejects credentials

Solutions:

  • Verify username and password spelling
  • Check Caps Lock status
  • Clear browser cache and cookies
  • Try different browser or incognito mode
  • Contact administrator if account is locked

‚è±Ô∏è Session Expires Quickly

Symptoms: Frequent logout prompts

Solutions:

  • Check system timeout settings with admin
  • Ensure cookies are enabled
  • Avoid opening multiple tabs
  • Request session length adjustment

Data & Display Issues

üì≠ Missing Data

Symptoms: Assets or records not visible

Solutions:

  • Check applied filters and search terms
  • Verify permissions for data access
  • Try refreshing the page (F5)
  • Check if items are archived
  • Contact admin to verify data integrity

üêå Slow Performance

Symptoms: Pages load slowly or timeout

Solutions:

  • Check internet connection speed
  • Clear browser cache and restart browser
  • Reduce number of displayed items per page
  • Contact admin about server performance
  • Try accessing during off-peak hours

Asset Management Issues

‚ùå Can't Checkout Asset

Symptoms: Checkout button disabled or fails

Solutions:

  • Verify asset status is "Available"
  • Check your user permissions
  • Ensure target user account is active
  • Verify asset isn't under maintenance
  • Confirm asset isn't already checked out

üîÑ Duplicate Asset Tags

Symptoms: Error when creating assets

Solutions:

  • Use the asset tag generator
  • Check existing assets before creating
  • Update duplicate tags immediately
  • Implement tag validation rules
  • Contact admin to resolve conflicts
